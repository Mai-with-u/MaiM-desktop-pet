## 🎯 架构设计

### 核心思想

**完全解耦**：每个协议独立处理自己的消息流，不需要全局协调

```
Maim 协议 (WebSocket)
├── 初始化：创建 Router
├── 连接：register_class_handler(_handle_incoming_message)
├── 发送：router.send_message()
└── 接收：_handle_incoming_message() → 触发信号 ✅

OpenAI 协议 (HTTP API)
├── 初始化：创建 session
├── 发送：HTTP 请求
└── 接收：send_message() 中直接处理响应 → 触发信号 ✅
```

## 🎯 最终架构

### Maim (WebSocket) 消息流

```
[启动]
初始化 → 创建 Router
    ↓
连接 → register_class_handler(_handle_incoming_message)
    ↓
[接收消息]
服务器推送 → _handle_incoming_message
    ↓
格式转换 + 保存数据库 + 触发信号
    ↓
UI 显示 ✅
```

### OpenAI (HTTP API) 消息流

```
[发送消息]
用户输入 → send_message()
    ↓
HTTP 请求 → API 响应
    ↓
_handle_response()
    ↓
保存数据库 + 触发信号
    ↓
UI 显示 ✅
```

---

## 📝 架构原则

1. **单一职责**：每个协议只负责自己的消息流
2. **独立初始化**：无外部依赖，各自初始化
3. **直接触发**：消息处理完成直接触发 UI 信号
4. **零协调**：不需要全局协调器或注册流程
